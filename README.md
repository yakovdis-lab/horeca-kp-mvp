# MVP: Персонализированное КП для HoReCa

Прототип веб-инструмента для менеджеров продаж.

## Что умеет

- Сценарий 1: подбор по одной или нескольким категориям клиента
- Сценарий 2: подбор по меню клиента (через файл соответствий `menu_item -> sku`)
- Сценарий 3: подбор по меню с исключением уже покупаемых позиций (по истории продаж)
- Фильтрация по складу (`stock_qty > 0`)
- Подстановка цены из прайса
- Подстановка ссылки на товар (`sku -> url`)
- Выгрузка КП в Excel: `Наименование товара`, `Цена товара`, `Ссылка`

## Быстрый старт локально

1. Установите Python 3.10+
2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Запустите приложение:

```bash
streamlit run app.py
```

## Входные файлы

Приложение принимает `xlsx/xls/csv`.

- `stock`: `sku`, `stock_qty`
- `price`: `sku`, `name`, `price`
- `sku_links`: `sku`, `url`
- `menu` (опционально): `menu_item`
- `menu_map` (для сценариев с меню): `menu_item`, `sku`
- `sales_history` (опционально): `sku`, `qty`
- Фиксированные файлы категорий: `data/category_assortment/*.csv` (колонка `name`)

Готовые шаблоны лежат в папке `templates/`.
Категории уже закреплены в папке `data/category_assortment/`:
`fastfood.csv`, `cafe.csv`, `bar.csv`, `restaurant.csv`, `coffeehouse.csv`, `retail.csv`, `tourism.csv`.

## Как разместить онлайн (прототип)

Самый простой путь: Streamlit Community Cloud.

1. Создайте репозиторий на GitHub и загрузите файлы проекта.
2. Откройте [share.streamlit.io](https://share.streamlit.io).
3. Нажмите `New app`.
4. Выберите репозиторий и файл `app.py`.
5. Нажмите `Deploy`.

Через 1-2 минуты получите публичную ссылку на прототип.

## Ограничения текущего MVP

- Разбор меню из PDF/JPEG пока не реализован (для MVP используется `menu.xlsx/csv`).
- Интеграция с `avistrade.by` пока через файл `sku_links`.
- Для лучшей точности названия в категориях должны совпадать с колонкой `name` в прайсе.
