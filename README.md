# MVP: Персонализированное КП для HoReCa

Прототип веб-инструмента для менеджеров продаж.

## Что умеет

- Сценарий 1: подбор по одной или нескольким категориям клиента
- Сценарий 2: автоматический разбор меню и подбор ингредиентов
- Сценарий 3: меню + история продаж (предлагает то, что клиент не покупает)
- Фильтрация по складу (`stock_qty > 0`)
- Подстановка цены из прайса
- Подстановка ссылки на товар (`sku -> url`)
- Выгрузка КП в Excel: `Наименование товара`, `Цена товара`, `Ссылка`

## Входные файлы

Приложение принимает `xlsx/xls/csv`.

- `stock`: `sku`, `stock_qty`
- `price`: `sku`, `name`, `price`
- `sku_links`: `sku`, `url`
- `menu` (для сценариев с меню): `menu_item`
- `sales_history` (опционально): `sku`, `qty`
- Фиксированные файлы категорий: `data/category_assortment/*.csv` (колонка `name`)
- Файл правил авторазбора меню: `data/menu_ingredient_rules.csv`
  - колонки: `dish_keyword`, `ingredient_name`

## Как настраивать авторазбор меню

Файл `data/menu_ingredient_rules.csv` работает так:
- если в названии блюда есть `dish_keyword`, то в КП добавляется `ingredient_name`
- затем `ingredient_name` сопоставляется с `price.name` и превращается в SKU

Пример:

```csv
dish_keyword,ingredient_name
пицца,Томаты резаные 2.5кг
пицца,Сыр моцарелла 1кг
```

## Ограничения MVP

- Это rule-based автоподбор (по ключевым словам), не полноценный ИИ-техкарт анализ.
- Для лучшей точности расширяйте `menu_ingredient_rules.csv`.
- Названия ингредиентов должны совпадать с колонкой `name` в прайсе.
